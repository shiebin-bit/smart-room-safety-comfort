Operating Logic

What the output does

AUTO mode

The dashboard follows sensor status. Relay turns on only during CRITICAL status and the buzzer follows alert settings.

MANUAL mode

User commands are sent to the ESP32 so relay and buzzer can be tested directly from the cloud dashboard.

Setup recovery

Enter Local Setup Mode asks the ESP32 to open its WiFi setup portal again if the network needs to be changed.

First-Time Setup

Pair a new ESP32

  1. Register or login with an invite code.
  2. Go to Devices and click Add Device.
  3. Copy the pairing code shown on the device card.
  4. Connect to SmartRoom-Setup-xxxx.
  5. Open http://192.168.4.1 and enter WiFi details plus pairing code.
  6. Reconnect phone or laptop to normal WiFi and wait for Online.
WiFi Management

Switch network remotely

  1. Device must already be Online.
  2. Click Scan Networks and wait for ESP32 check-in.
  3. Select a network, enter password and connect.
  4. Use Reconnect on saved profiles when the password is already stored on ESP32.
  5. Use Enter Local Setup Mode only for recovery.
Security

How access is protected

  • Registration requires an invite code.
  • Each user only sees their own devices.
  • Pairing codes expire and are cleared after pairing.
  • ESP32 uses a hidden device token after pairing.
  • Saved WiFi passwords stay on the ESP32, not in SQLite.
Troubleshooting

Common checks

PAIR 404
Pairing code is wrong, expired or already used. Generate a new device code.
PAIR 530
Cloudflare tunnel or Flask backend is not reachable. Check server and tunnel status.
No data
Check Serial Monitor for POST status and confirm the device is Online.
WiFi scan empty
Wait for the next device check-in after requesting Scan Networks.
